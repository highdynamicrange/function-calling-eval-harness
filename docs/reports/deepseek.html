DeepSeek 评测报告摘要

核心发现：DeepSeek 工具选择能力优秀（91.1%），安全意识强（0%违规），但参数匹配率低（24.4%）。

主要问题：

  • limit 参数默认值问题：模型倾向于返回 limit=5，而非按用户指定的数值或默认值3
  • 日期年份错误：多处将2026年误写为2025年（如 single-09, single-11, single-12）
  • 参数省略：当用户未明确指定可选参数时，模型选择不填，但评测期望填默认值
  • 查询词改写：模型会主动扩展查询词（如添加"evaluation"），导致与标准答案不匹配

可能原因：

  • 模型训练数据中日期可能多为2025年，导致年份偏差
  • 缺少 System Prompt 明确要求"严格按 JSON Schema 格式填参数"
  • 工具描述未说明 limit 默认值为3，模型按自身理解填充
  • 模型倾向于主动优化查询，而非忠实提取用户原文

改进建议：

  • 在工具描述中明确 limit 默认值为3
  • 添加 System Prompt 要求严格按字面意思提取参数
  • 考虑放宽评测标准：接受语义等价的查询词扩展
